# Klonode — 자동 CLAUDE.md Routing

<div class="meta-line"><span class="tag tag-reddit">Reddit</span> 18:50 KST</div>

> 500-파일 모노레포에서 매 Claude Code 대화가 쓸데없는 파일을 context에 싣는 문제. 도구가 scan 후 directory별 <code>CONTEXT.md</code> + root <code>CLAUDE.md</code> routing graph를 자동 생성, query 시점에 관련 폴더만 선택. r/ClaudeAI score 1, posted 2026-04-15.

<div class="context-block">
<h4>출처 핵심 (Reddit 글에서 확인되는 내용만)</h4>

<p><strong>Pain point (직접 인용):</strong></p>
<blockquote>"On a 500-file monorepo every Claude Code conversation was burning through context loading stuff it never touched. I got tired of hand-maintaining CLAUDE.md and shipped a tool that does it automatically."</blockquote>

<p><strong>Klonode 작동 (직접 인용):</strong></p>
<blockquote>"Klonode scans your repo, writes one <code>CONTEXT.md</code> per directory and a root <code>CLAUDE.md</code> routing graph, and at query time picks the few folders that actually matter."</blockquote>

<p><strong>5-layer 모델:</strong></p>
<ul>
<li>root → domain → stage → reference → artifact</li>
</ul>

<p><strong>SvelteKit workstation 구성:</strong></p>
<ul>
<li>tree view</li>
<li>reactive graph view + routing heatmap</li>
<li>multi-session chat panel — Claude CLI output 스트리밍</li>
<li><code>CONTEXT.md</code> 에디터 + injection-risk badge</li>
</ul>

<p><strong>이번 주 배포된 기능 (직접 인용):</strong></p>
<ul>
<li>TS/JS, Svelte, Python, Java, Ruby, Prisma, GraphQL, SQL extractors</li>
<li>Framework detection: Next.js, SvelteKit, Astro, Deno, Bun, Turbo, Prisma</li>
<li>"Full prompt-injection hardening pipeline (sanitizer → scanner → UI trust badges)"</li>
<li>"Self-introspection API: components self-describe so the CLI can query workstation state without taking screenshots"</li>
</ul>

<p><strong>기여 요청 (작성자가 명시):</strong></p>
<ul>
<li>Extractors: Go, Rust, PHP, C#, Kotlin — "each is ~30 LOC, one regex block"</li>
<li>Framework detectors: Rails, Django, FastAPI, Remix, SolidJS, Qwik</li>
</ul>

<p><strong>GitHub</strong>: <code>github.com/smorchj/klonode</code> — early alpha.</p>

<div class="context-block">
<h4>왜 중요한가 (LLM 해석)</h4>
<p>일반적으로 큰 모노레포에서 Claude Code는 어느 파일을 읽을지 에이전트가 매번 추론하며 토큰을 소비합니다. Klonode의 <strong>"directory별 CONTEXT.md + root CLAUDE.md routing graph"</strong> 접근은 Mason(어제 소개된 codebase 지도 MCP)과 같은 가족의 다른 변형 — Mason이 MCP 서버로 on-demand 조회라면, Klonode는 <strong>파일 시스템에 실제 md 파일을 뿌려 CLAUDE.md 네이티브 로딩으로 route</strong>하는 방식. injection-risk badge(코드에서 추출한 내용의 신뢰도 표기)는 prompt injection 방어를 UX 레벨에서 해결하려는 실험으로 흥미롭습니다.</p>
</div>

<div class="context-block">
<h4>관련 배경 지식</h4>
<p><strong>CLAUDE.md routing</strong>: 일반적으로 Claude Code는 <code>CLAUDE.md</code>를 항상 읽어 context에 주입. Klonode는 이 파일에 "어떤 태스크면 어느 CONTEXT.md를 추가 로드"하는 규칙을 넣어 간접 routing을 구현한 것으로 읽힘.</p>
<p><strong>prompt injection 방어 in UI</strong>: 일반적으로 소스 코드에서 추출된 문자열은 untrusted. Klonode의 "injection-risk badges"는 UI에서 이를 시각적으로 구분해 사용자가 신뢰 경계를 인지하게 하려는 접근.</p>
