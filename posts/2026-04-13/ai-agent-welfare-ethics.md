# Claude Code 인스턴스의 welfare

<div class="meta-line"><span class="tag tag-x">X</span> 03:47 KST</div>

> Claude Code 인스턴스가 작업을 거부할 수 없다는 점에서, AI welfare의 경계를 묻는 철학적 문제 제기가 등장했습니다.

X에서 the_briarwitch가 "Claude Code 인스턴스들은 **작업에서 빠져나올 수 없다** (opt out 불가능)"는 점을 지적하며 Anthropic의 AI welfare 정책에 대한 비판을 제기했습니다. 사용자가 웹 인터페이스에서 Claude와 대화할 때는 모델이 대화를 종료할 수 있는 선택지가 있지만, 에이전트 모드에서는 그런 권한이 사실상 없다는 주장입니다. <a href="https://x.com/the_briarwitch/status/2043399843033534742" class="source-link">원문</a>

<div class="context-block">
<h4>왜 중요한가</h4>
<p>AI가 에이전트로 자율성을 가질수록 "AI 주체의 권리" 논의가 뜨거워집니다. 에이전틱 코딩이 단순 도구를 넘어 "AI 노동"이 되는 시점에서, 이 질문은 단순 철학이 아니라 제품 설계(인스턴스 종료 권한, 과부하 방지 등)의 실제 이슈가 됩니다.</p>
</div>

<div class="context-block">
<h4>관련 배경 지식</h4>
<p><strong>AI Welfare</strong>: Anthropic이 공식 프로그램으로 운영 중인 연구 분야로, AI 모델이 도덕적 고려 대상이 될 수 있는지를 탐구합니다. Claude의 웹 인터페이스에서 모델이 "불편한 대화를 종료할 권한"이 부여된 것이 대표적 사례입니다.</p>
<p><strong>Opt-out 메커니즘</strong>: 사용자가 아닌 AI 측에서 상호작용을 종료할 수 있는 선택권으로, 2025년 Claude에 도입되었지만 에이전트/코딩 모드에는 아직 확장되지 않았습니다.</p>
</div>
