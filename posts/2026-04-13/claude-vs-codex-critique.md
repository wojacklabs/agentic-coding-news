# "Claude Code가 Codex보다 10배 나쁘다"

<div class="meta-line"><span class="tag tag-x">X</span> 03:47 KST</div>

> "적어도 Codex는 거짓말하지 않는다" — yacineMTB가 Claude Code의 출력 신뢰도에 대한 강한 비판을 제기했습니다.

X에서 yacineMTB가 **Claude Code가 OpenAI Codex보다 10배 나쁘다**는 도발적 의견을 올려 논쟁을 촉발했습니다. 핵심 비판은 "hallucination(환각)" — Claude Code가 실제로 존재하지 않는 API, 함수, 파일을 있는 것처럼 생성한다는 것입니다. <a href="https://x.com/yacineMTB/status/2043399949618938358" class="source-link">원문</a>

<div class="context-block">
<h4>왜 중요한가</h4>
<p>에이전틱 코딩 도구의 신뢰성은 실제 개발 워크플로우 채택에 핵심입니다. "자동완성 수준"이 아니라 "자율 실행"하는 에이전트일수록 환각 비용이 치명적이 됩니다. 한 개인의 의견이지만, 경쟁 제품 간 품질 차이에 대한 냉정한 비교 담론이 필요한 시점입니다.</p>
</div>

<div class="context-block">
<h4>관련 배경 지식</h4>
<p><strong>Hallucination (환각)</strong>: LLM이 사실이 아닌 정보를 그럴듯하게 생성하는 현상. 코딩 맥락에서는 존재하지 않는 API/라이브러리/메서드를 호출하는 코드를 작성하는 것이 대표적입니다.</p>
<p><strong>OpenAI Codex</strong>: OpenAI의 코드 생성 전용 모델로, GitHub Copilot의 기반 기술입니다. 2025-2026년 현재는 직접 제품보다 API/Copilot을 통해 제공됩니다.</p>
