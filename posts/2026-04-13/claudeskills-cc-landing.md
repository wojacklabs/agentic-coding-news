# claudeskills.cc — 주의: Anthropic 아님

<div class="meta-line"><span class="tag tag-hn">HN</span> 19:02 KST</div>

> 이름과 달리 Anthropic 공식 제품이 아니며, Claude·GPT·Gemini·Llama 등 다중 모델용 Skill을 만드는 제3자 상용 플랫폼 "Agent Skill Studio"의 얼리액세스 랜딩 페이지.

<div class="context-block">
<h4>출처 핵심 (claudeskills.cc 랜딩 페이지에서 확인되는 내용만)</h4>
<p><strong>사이트 이름·성격:</strong></p>
<ul>
<li>사이트 표제: <strong>Agent Skill Studio</strong></li>
<li>명시된 고지: "not affiliated with, endorsed by, or sponsored by Anthropic, Claude AI, or any related organizations"</li>
<li>유형: 제품 랜딩 페이지 + 얼리액세스 플랫폼 (공개 마켓플레이스나 오픈 레지스트리 아님)</li>
</ul>

<p><strong>기능 주장:</strong></p>
<ul>
<li>"create reusable Agent Skills for any AI model"</li>
<li>"deploy seamlessly across Claude, GPT, Gemini, Llama, and emerging AI platforms"</li>
<li>버전 관리, audit-ready history, 엔터프라이즈 권한, skill 협업 워크플로우</li>
</ul>

<p><strong>프리빌트 템플릿 (마켓플레이스 목록이 아닌 샘플):</strong></p>
<ul>
<li>Customer support triage automation</li>
<li>Research brief generation</li>
<li>Document automation</li>
<li>Product operations co-pilot</li>
</ul>

<p><strong>랜딩에 표시된 마케팅 지표:</strong></p>
<ul>
<li>"10+ AI models supported"</li>
<li>"3x faster" (아이디어 → 프로덕션)</li>
<li>"100% Model portability"</li>
</ul>
</div>

<div class="context-block">
<h4>왜 중요한가 (LLM 해석)</h4>
<p>Anthropic이 Agent Skills를 공식 런칭한 당일 HN에 올라온 제3자 제품으로, 이름 부여(<code>claudeskills.cc</code>)만 놓고 보면 공식 레지스트리로 오해할 여지가 큽니다. 실제로는 Claude와 무관한 상용 플랫폼이며 고지문에 무관함을 명시해 둔 상태입니다. 일반적으로 브랜드명 지명도를 활용한 제3자 사이트는 커뮤니티가 주의해서 해석할 필요가 있으며, Skills 생태계가 초기 단계라는 점을 감안하면 향후 유사 혼동 사례가 늘어날 것으로 보입니다.</p>
</div>

<div class="context-block">
<h4>관련 배경 지식</h4>
<p><strong>"Skill portability"의 의미 차이(일반)</strong>: Anthropic 공식 Skills는 "Claude Apps · Claude Code · API 간 포팅"을 포터빌리티로 정의합니다. Agent Skill Studio가 말하는 "100% Model portability"는 서로 다른 모델 벤더(Claude·GPT·Gemini 등) 간 호환을 뜻합니다 — 유사 용어지만 범위가 다릅니다.</p>
