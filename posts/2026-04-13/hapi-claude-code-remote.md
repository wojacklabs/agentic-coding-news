# HAPI — 로컬 세션 + 원격 제어

<div class="context-block">
<h4>출처 핵심 (트윗에서 확인되는 내용만)</h4>

<p><strong>트윗 본문 (직접 인용):</strong></p>
<blockquote>"HAPI enables the launch of Claude Code, Codex, or Gemini sessions locally and their remote control via web, mobile, or Telegram, with seamless handoff, AFK validations, and end-to-end encrypted relay."</blockquote>

<p><strong>위 본문에서 확인되는 기능 4가지:</strong></p>
<ul>
<li>로컬 세션 launch (Claude Code/Codex/Gemini)</li>
<li>web/mobile/Telegram 원격 제어</li>
<li>seamless handoff</li>
<li>AFK validations</li>
<li>end-to-end encrypted relay</li>
</ul>

<div class="context-block">
<h4>왜 중요한가 (LLM 해석)</h4>
<p>일반적으로 Claude Code는 로컬 터미널에서 실행되며, 외출 중에는 접근이 어렵습니다. HAPI 같은 원격 제어 패턴은 핸즈프리 워크플로우 시나리오에서 반복적으로 등장하는 주제로, Telegram·iMessage·웹 채널을 통한 에이전트 지시·결과 수신은 작년부터 활성화된 영역입니다. 단, 이 트윗만으로는 HAPI의 구체 구현·라이선스·실제 기능 수준은 확인할 수 없으므로 도입 전 별도 자료 확인이 필요합니다.</p>
</div>

<div class="context-block">
<h4>관련 배경 지식</h4>
<p><strong>AFK validations</strong>: 일반적으로 "Away From Keyboard" 시나리오에서 사용자에게 확인을 요청하는 메커니즘. 에이전트가 위험한 작업 직전에 사용자 모바일로 승인 요청 보내는 패턴이 흔합니다.</p>
<p><strong>End-to-end encrypted relay</strong>: 일반적으로 서버가 평문을 보지 않도록 클라이언트 간 직접 암호화 채널을 의미. 에이전트 명령·결과의 프라이버시 보장 목적.</p>
