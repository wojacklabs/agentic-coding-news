# /powerup — 18 interactive lessons

<div class="meta-line"><span class="tag tag-x">X</span> 09:35 KST</div>

> Claude Code에 <code>/powerup</code> 슬래시 명령이 추가됐다는 X 보고. 터미널 안에서 18개의 인터랙티브 레슨이 애니메이션과 함께 최신 기능을 시연.

<div class="context-block">
<h4>출처 핵심 (트윗에서 확인되는 내용만)</h4>

<p><strong>트윗 본문 (직접 인용):</strong></p>
<blockquote>"Claude Code has added /powerup. 18 interactive lessons run within the terminal. They demo the latest features with animations. There were so many updates that I'd missed some features, but I found them all here. It's worth typing it once at least."</blockquote>

<p><strong>정리된 사실:</strong></p>
<ul>
<li>슬래시 명령: <code>/powerup</code></li>
<li>레슨 개수: 18개</li>
<li>실행 위치: 터미널 내부</li>
<li>내용: "latest features" 데모 + 애니메이션</li>
<li>작성자 평: "worth typing it once at least"</li>
</ul>

<p><strong>참고</strong>: 트윗에는 추가된 Claude Code 정확한 버전, 각 레슨의 구체 제목·길이, 공식 문서 URL 등은 포함되지 않음.</p>
</div>

<div class="context-block">
<h4>왜 중요한가 (LLM 해석)</h4>
<p>일반적으로 Claude Code는 잦은 업데이트로 기능이 늘어나는데, 사용자가 모든 변경사항을 릴리스 노트로 따라가기는 어렵습니다. <code>/powerup</code> 같은 <strong>"in-terminal 온보딩/튜토리얼 커맨드"</strong>는 이미 세션 안에 있는 사용자에게 설치 없이 기능 학습 기회를 제공한다는 점에서 사용성 측면 개선으로 보입니다. 팀에서 신규 합류자 온보딩에 재활용할 수 있는 기본 자료가 됩니다.</p>
</div>

<div class="context-block">
<h4>관련 배경 지식</h4>
<p><strong>Claude Code 슬래시 명령</strong>: 일반적으로 <code>/help</code>, <code>/config</code>, <code>/stats</code> 등 빌트인 명령이 있으며, 사용자가 <code>.claude/commands/</code>에 커스텀 명령을 만들어 확장할 수 있습니다. <code>/powerup</code>은 빌트인으로 추가된 것으로 트윗은 읽히지만, 빌트인 여부 자체는 트윗 본문에서 명시적으로 확인되지 않습니다.</p>
