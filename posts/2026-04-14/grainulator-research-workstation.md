# grainulator — Claude Code Research Plugin

<div class="context-block">
<h4>왜 중요한가 (LLM 해석)</h4>
<p>일반적으로 Claude Code를 "코딩 도구"를 넘어 "리서치 워크스테이션"으로 쓰는 패턴은 Obsidian 연동, subagents 병렬 관점, MCP 서버 통한 자료 접근 등 여러 갈래가 있습니다. grainulator는 이 방향성 중 하나를 플러그인 형태로 구조화한 사례로 보이며, 비슷한 시나리오(자료 수집 → 분석 → 결론 정리 파이프라인)를 Claude Code 안에 고정하고 싶은 사용자에게 참고 가치가 있을 수 있습니다. 구체 기능은 원 GitHub 확인이 필요합니다.</p>
</div>

<div class="context-block">
<h4>관련 배경 지식</h4>
<p><strong>Claude Code 플러그인</strong>: 일반적으로 Anthropic의 plugin 메커니즘은 marketplace 기반 설치와 <code>.claude/</code> 구성 파일 방식 등 여러 경로가 알려져 있습니다. grainulator가 어떤 방식을 쓰는지는 이 기사 출처 범위에서 별도 검증하지 않았습니다.</p>
